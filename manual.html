Getting Oriented

How Your Website Works

Your entire website is just a small handful of files sitting together in one folder. There's no database, no subscription CMS, no server software — which means nothing to break, nothing to pay monthly for, and nothing that can lock you out.

Where you make edits.
→exports
🧠
config.js
The memory.
Every word, photo path, color & setting.
→feeds
🌐
index.html
The website.
What your visitors see.
Plus one small extra: favicon — the tiny icon shown in the browser tab.

The four files, in plain English

FileWhat it isDo you ever edit it?
index.htmlThe website itself — the page your visitors open. It reads all of its text, images, and colors from config.js.Never. Leave it alone.
admin.htmlYour private control panel. Open it in a browser to change anything about the site through friendly forms.You use it, but never edit the file itself.
config.jsThe site's memory. A single file holding all of your content and settings. The admin creates fresh copies of this for you.Only by replacing it with a new copy exported from the admin.
faviconThe little icon in the browser tab (usually favicon.ico or favicon.png).Only if you want a new tab icon — just swap the image file.
The one idea to remember

The admin does not change your live website directly. It saves your work inside your browser, then lets you download a fresh config.js. Your website only changes when you replace the old config.js in your site folder with the new one. This is covered step-by-step in Publishing Changes.

The edit → preview → publish cycle

  1. Edit — open admin.html in your browser and make changes. Everything autosaves to the browser as you type (watch for “Autosaved” in the top bar).
  2. Preview — click 💾 Save & Preview in the top bar. The site opens in a new tab showing your changes. Only you can see this preview — it lives in your browser.
  3. Publish — happy with it? Click ⬇ Export config.js, then put the downloaded file into your website folder, replacing the old one. Now the whole world sees it.

Folder Structure

Keeping the folder tidy isn't just about neatness — the admin refers to images and documents by their path (their address inside the folder), so a predictable structure makes every image field easy to fill in.

my-website/ ├── index.html ← the website (never edit) ├── admin.html ← the control panel (never edit) ├── config.js ← the content (replace when publishing) ├── favicon.png ← browser tab icon ├── images/ ← all photos, logos & graphics │ ├── logo.png │ ├── logo-alt.png │ ├── hero.jpg │ └── team-jane.jpg └── files/ ← downloadable documents (PDFs etc.) └── intake-form.pdf

How paths work

When an admin field asks for an “Image Path,” you type the file's address relative to the site folder. If your photo lives at images/hero.jpg, that's exactly what you type — lowercase, with the folder name, separated by a forward slash.

  • Match capitalization exactly. On most web hosts, Hero.JPG and hero.jpg are different files. Sticking to all-lowercase filenames avoids this trap entirely.
  • Avoid spaces in filenames. Use dashes instead: team-photo-2026.jpg, not Team Photo 2026.jpg.
  • Web addresses work too. Any field that takes a path also accepts a full URL like https://example.com/photo.jpg, but files stored in your own images/ folder are more reliable — outside links can vanish.
  • Adding a new image is a two-part job: copy the image file into images/ using your file explorer, then type its path into the admin field. Many admin fields show a small live preview when the path is correct.
Good habit

Name images by what they are: images/hero-office.jpg, images/logo-white.png, images/gallery-kitchen-01.jpg. Six months from now, you'll thank yourself.

Backups & Archives

Here's the good news: because your entire site's content lives in one file (config.js), backing up is almost embarrassingly easy. There are two levels of backup, and you should use both.

Level 1 — Content backups (do this often)

Every time you finish a round of edits, export config.js and keep a dated copy in a backup folder:

my-website-backups/ ├── config-2026-07-01.js ├── config-2026-07-10.js └── config-2026-07-15.js ← today's export, renamed with the date
  1. Export — in the admin, click ⬇ Export config.js. The file lands in your Downloads folder.
  2. Rename — add today's date to the copy you're archiving, e.g. config-2026-07-15.js.
  3. Store — move it into a backup folder that lives somewhere safe — ideally a cloud-synced folder (Google Drive, Dropbox, OneDrive) so a broken computer can't take it with it.
Why this works

Restoring from a backup takes seconds: open the admin, drag the backup file onto the “Drag & drop your config.js here” zone at the top, and the entire site state from that date loads back in. Then export and publish as usual.

Level 2 — Full-site archives (do this occasionally)

Once in a while — and always before big changes — archive the whole folder, including images and documents:

  1. Right-click your website folder in your file explorer.
  2. Compress it — choose “Compress to ZIP file” (Windows) or “Compress” (Mac).
  3. Date it and store it — rename to something like my-website-2026-07-15.zip and keep it with your backups. This single ZIP is a complete, restorable copy of your entire website.
What autosave does NOT protect you from

The admin's autosave lives inside one browser on one computer. Clearing your browser data, switching computers, or a hard drive failure will erase it. Your exported config.js files are the only backups that truly count — export early, export often.

Using the Admin

Admin Basics

Open admin.html in any modern browser (Chrome, Edge, Firefox, Safari). You'll see three areas: a dark top bar, a dark sidebar on the left, and the editing area in the middle.

The top bar

The top bar: status, Save & Preview, and Export config.js.
ControlWhat it does
Status textShows “Autosaved 3:42 PM” style messages. If you've typed something and this hasn't updated within a couple of seconds, pause before closing the tab.
💾 Save & PreviewSaves everything and opens the website in a new tab so you can see your changes. This preview is private to your browser — visitors still see the old site.
⬇ Export config.jsDownloads the file that makes changes real. See Publishing Changes.

The import zone

The import zone at the top of the editing area.

At the top of the editing area sits a dashed box: “Drag & drop your config.js here to import existing settings.” Use it when:

  • Starting a session on a new computer or browser — drag in your current live config.js so the admin picks up where the site actually is.
  • Restoring a backup — drag in any dated backup to roll back to that moment.
Rule of thumb

If you're ever unsure whether the admin is showing the latest version of your site, import the config.js that's currently sitting in your website folder. That file is always the source of truth for what's live.

Panels, switches, and repeating items

  • Panels — white cards with a title bar. Click the title bar to open or collapse a panel. Collapsing is just for tidiness; your data is safe either way.
  • Toggle switches — sliding on/off switches. Some hide entire pages (e.g., the About Us page toggle); when off, that page disappears from navigation and visitors are redirected home. Your content is kept — flip it back on any time.
  • Repeating items — anywhere you see a + Add … button (stats, team members, locations, cards…), you can add as many items as you like. Each item has ↑ ↓ arrows and a ⋇ drag handle to reorder, and a × to remove it.
  • Color fields — offer a color-picker square, a swatch button that suggests colors from your palette, and a text box for exact hex codes (like #c9a84c) if you have brand colors.
  • On phones and tablets — the sidebar hides behind a floating ☰ Menu button in the corner.
Deleting is instant

Removing an item, section, or page takes effect immediately — there is no undo button. If you're about to delete something substantial, export a config.js backup first, or use the section's Export JSON button (see How Sections Work) to save just that piece.

Using the Admin

Sidebar Tour

The sidebar is your map. It's organized into three groups — Content (the pages themselves), Business Info (facts about your organization), and Design (site-wide look, behavior, and publishing). Here's what lives in each entry.

🏛️ Business Details

The core identity used all over the site.

  • Business Identity — business name, tagline/specialty, the tagline shown on the launch screen, a short footer bio (1–2 sentences), and an optional custom copyright line. Also holds the Google Maps API Key, which Location sections need for embedded interactive maps (leave blank to use a “open in Google Maps” link instead).
  • Branding & Logo — paths for a Primary and an optional Alternate logo (typically a light version for dark backgrounds), plus separate controls for which logo appears in the header, on the splash screen, and in the footer, each with its own height.

📞 Contact & Hours

Everything on the Contact page, plus a master switch to hide the page entirely.

  • Page Header — the label, headline, and subline at the top of the Contact page.
  • Locations — add one card per office/branch with address, phone, hours, and map settings. Reorder freely.
  • Contact Form — heading, intro, privacy note, submit-button label, and success message. The Formspree Endpoint is where submissions get emailed to you: create a free form at formspree.io and paste the URL it gives you. Below that is a form field builder — add, reorder, and configure every field on the form (text, email, phone, dropdowns, message boxes; mark fields required or half-width).
  • Social Media — add links that appear in the footer.

🔍 Search

Visitors can search your whole site from the magnifying-glass icon in the header. Here you set the placeholder text, the “no results” messages (use {query} to echo back what they typed), results per page, and — under Excluded Pages — toggle off any pages you'd rather keep out of search results (useful for members-only pages). Two more switches: Show “Clear” Button on Recent Searches lets visitors clear their own saved search history (remembered per-browser, not shared across visitors), and Highlight Result on Page briefly outlines the matched text after someone clicks a result and lands on that section, so it's obvious what the search actually found.

🌅 Splash / Launch Screen

The brief branded screen shown while the site loads. Choose which logo appears (or none, showing the site name as text), its size, and a display mode (logo + name + tagline, logo + tagline, or logo only). The background beneath it can be a solid color, a gradient texture, a large faint watermark logo (with size and opacity sliders — even a custom image), or a full photo with a color overlay to keep the text readable.

🏠 Home Page

Three stacked parts:

  • Hero Section — the full-screen banner at the top. Headline & Text: the headline (commas split it into separate animated lines), subline, an optional live search bar (with its own placeholder text) that sits right in the hero below the subline, trust bullets (one per line), and two buttons with smart links. Background: pick Solid Color, Single Image, Slideshow (multiple slides with crossfade/slide transitions, duration, autoplay, dots), or Video (a playlist of MP4 clips with a shared fallback image for mobile and an optional “hold on fallback” pause between loops). All backgrounds get optional Ken Burns motion (zoom/pan with a speed slider) and a color overlay with opacity to keep the text legible.
  • Stats Bar — the row of numbers under the hero (e.g., “20+ Years”). Add stats, reorder, and toggle count-up animation.
  • Page Sections — the full section builder for everything below the stats bar. Works identically to custom pages — see How Sections Work.

👥 About Us

A page toggle, a rename field (change “About Us” to “Our Team,” “Meet the Doctor,” anything — it updates the navigation everywhere), banner label/title, and then team member profiles: add as many people as you need, each with name, credentials, photo, and biography. Below the profiles, an Additional Sections builder lets you add any section type to extend the page.

🎨 Colors & Fonts

The site-wide visual identity, with a live preview that updates as you pick.

  • Section Backgrounds — the Light, Alt, and Dark background colors that every section on the site chooses from, plus a lighter dark-variant used for hovers and gradients.
  • Accent Colors — the main accent (buttons, labels, highlights), its hover shade, and a subtle pale tint.
  • Text Colors — heading and body text colors.
  • Chrome — the header/footer bar color only; it never affects section backgrounds.
  • Typography — pick a display font (headlines) and body font from curated lists, or type any font name from Google Fonts into the custom override fields (spelling and capitalization must match exactly). A preview box shows the pairing live.

🧭 Navigation

Behavior switches for the menus: whether groups of subpages start expanded in the main nav and footer nav, and whether the header is always solid (instead of starting transparent over the hero). Also home of the Nav/Footer CTA button — the prominent call-to-action button in the header and footer, with a label and smart link.

🔍 SEO & Meta

How your site appears to search engines: the meta description (aim for 150–160 characters), keywords, an optional Google Analytics ID (looks like G-XXXXXXXXXX), and individual browser-tab titles for the Home, About, Contact, and Privacy pages. Custom pages set their titles in their own settings.

🔐 Members & Access

PIN-protect the whole site or individual pages.

  • Site Access — Public, or PIN Required (the PIN pad appears when a visitor opens the menu or scrolls). Session duration controls how many days a member stays logged in.
  • PIN Pad Appearance — every word on the lock screen is editable: headline, subtext, wrong-PIN message, max attempts, theme (Dark/Gold/Light), the lock icon shown next to protected pages, a personalized welcome screen ({name} inserts the member's name), and an optional “imposter” screen after too many failed attempts.
  • Members — add each member with their own unique PIN; the site greets them by name.
  • Page Access — per-page dropdown: public, or locked. Locked pages show the lock icon in the nav for guests.

📤 Export

Preview the generated config.js as text, download it, and read the built-in deployment reminder. Full walkthrough in Publishing Changes.

Building the Site

Pages & Navigation

Beyond the built-in Home, About, and Contact pages, you can create unlimited custom pages. Each becomes an entry in the sidebar and in your site's navigation menus.

Creating a page

  1. Click “＋ Add page” at the bottom of the Content group in the sidebar. A new page appears instantly.
  2. Open its Page Settings panel and give it an emoji (its nav icon), a Navigation Label (e.g., “Estate Planning”), and confirm the URL slug.
  3. Add sections using the builder below — that's the entire body of the page. See How Sections Work.

Page settings explained

The Page Settings panel — visibility, slug, hierarchy, and access.
SettingWhat it does
Show in navigationThe master switch. Off = the page is hidden from all menus and visitors who try its address are sent home. Content is kept.
Page EmojiThe little icon shown beside the page name in menus and the admin sidebar.
Navigation LabelThe page's name as shown in menus.
URL SlugThe page's web address ending, e.g. estate-planning. It's generated automatically from the label, but you can set it manually.

Careful with slugs

Changing a page's URL slug after your site is live breaks any links to the old address — bookmarks, links you emailed out, search engine results. Settle on slugs before you promote the page, and avoid changing them afterward.

Ordering and nesting

  • Reorder pages by grabbing the drag handle next to a page's name in the admin sidebar and dropping it where you want. Moving a parent brings its subpages along with it.
  • Subpages reorder among their siblings the same way.
  • Deleting a page — the × Delete Page button at the top of the page's panel removes it and all of its sections. Export a backup first if in doubt.
Building the Site

How Sections Work

Sections are the building blocks of every page. A page is simply a stack of sections, top to bottom — a text block, then a card grid, then a gallery, then a call-to-action banner. There are over 30 section types (fully catalogued in the Section Library), but they all share the same controls, so once you've configured one, you can configure any of them.

Adding a section

At the bottom of any page's section list you'll find a dropdown of section types, a + Add Section button, and an ⊕ Import button. Pick a type, click add, and the new section card appears at the bottom of the list — click its header to expand and fill it in.

Pick a type, then Add Section — or Import a saved one.

Smart links — used everywhere

Buttons, cards, and list items across nearly every section type share one linking system. Wherever you see a Link Type dropdown:

  • None — no link (buttons are hidden).
  • Internal Page — pick any page of your own site from a dropdown, plus an optional Scroll to Section target (populated from the Anchor IDs you've set).
  • External URL — any web address, e.g. https://example.com.
  • File / Document — a file from your files/ folder, e.g. files/intake-form.pdf.

Import / Export sections

The ⊕ Import button next to “Add Section” accepts a section JSON file exported earlier. Together with Export JSON this gives you a personal library of reusable, pre-styled sections: export your best CTA banner once, import it onto every new page in seconds.

Writing text in sections

In multi-paragraph text areas, press Enter twice (a blank line) to start a new paragraph. Most body fields also accept simple HTML if you know it, but you never need it — the Rich Text Block section offers bold, italics, lists, and headings with toolbar buttons.

Section Library

Customizable Page Sections

Every page on your site — Home, About Us, and every custom page you create — is just a stack of sections, one after another. There's no separate "homepage builder" or "custom page builder" hiding underneath; it's the same 33 section types and the same shared controls everywhere, which means what you learn in this chapter works identically no matter which page you're standing in.

That makes this the workhorse chapter of the whole manual. Business Details and Design get set up once and mostly left alone; sections are what you'll open the admin for again and again — adding a new service, updating a gallery, building an entire new page from scratch. Knowing this reference well is what turns "I want to add something to the site" into a five-minute task instead of a guessing game.

How to use this chapter

You don't need to read it start to finish. Each of the 8 groups below opens with a short writeup — skim those first to find the family of section that fits what you're trying to add. Then jump to that specific type, click "Fields & Controls" to expand its field list, and use it as a checklist while you build. Bookmark this page (or the sidebar link) and come back to it any time you're reaching for a new section.

🗒️ Content & Text

4 types

This is where most pages start and often where they end — plain writing, well presented. Reach for this group whenever the content is fundamentally a paragraph or two: an explanation, a policy, an origin story, a detailed answer to "what does this actually involve." The four types differ mainly in how much visual structure they add: Text Block is bare prose, Two Column pairs it with a photo, Rich Text adds real formatting for longer pieces, and Hero Section turns an opening statement into a full banner moment. A good rule of thumb: if you're tempted to write more than 3–4 sentences inside a card or bullet item elsewhere in the admin, that content probably belongs here instead.

🃏 Cards & Grids

5 types

Use this group when you have several comparable things — services, packages, differentiators, plans — and want a visitor to scan and compare them at a glance rather than read them in sequence. The five types trade off information density against interactivity: Feature Cards and Services Grid work well for "here's what we offer" overviews (Services Grid is the better pick when each item needs a longer explanation, since the detail hides in a popup until clicked); Tier Cards and Flip Cards both suit pricing or package comparisons, with Flip Cards adding a bit of showmanship via the hover reveal; Rotating Cards is the odd one out — a slow-moving carousel that's better for a rotating gallery of projects, testimonials, or team highlights than for anything a visitor needs to compare side by side. Keep card counts consistent with your column count (3 cards in 3 columns, not 4) for a tidy grid.

📐 Page Structure

3 types

These aren't content types so much as connective tissue — small, quiet tools for directing a visitor to the next thing you want them to do. CTA Banner is the loud, deliberate version: a full-width moment that says "here's the one thing to do next," and works best used sparingly — once per page, usually near the end. Button and Link are their understated cousins: rather than forming a section of their own, they attach quietly to the section above, letting you add a call-to-action to a Text Block or Rich Text piece without switching that section's type or breaking its visual flow. A well-built page usually threads a Button or Link after two or three content sections, then closes with one CTA Banner — guiding without nagging.

Going Live

Publishing Changes

Previewing shows changes to you. Publishing shows them to everyone. It's a three-step ritual — the same every time:

  1. Preview first. Click 💾 Save & Preview and click through the site in the new tab. Check the pages you changed — and glance at the ones you didn't.
  2. Export. Click ⬇ Export config.js (top bar, or the Export page in the sidebar). A fresh config.js lands in your Downloads folder.
  3. Replace. Move that file into your website folder, replacing the existing config.js. When asked whether to overwrite — yes. If your site is hosted online, upload the new config.js to your host the same way your site was originally uploaded (this replaces the copy on the server). Reload the site: your changes are live.
Nothing else needs uploading

For everyday content edits, config.js is the only file that changes. The exceptions: if you added new images or documents, those files must also be copied into the site's images/ or files/ folders (and uploaded to your host, if hosted online).

Make it a backup at the same time

You already have the exported file in hand — copy it into your backup folder with today's date before you move on. Publishing and backing up become one habit.

Going Live

Troubleshooting

SymptomLikely cause & fix
“My changes aren't on the live site”The #1 question — you previewed but didn't publish. Previews live only in your browser. Export config.js and replace the one in your site folder (and on your host). See Publishing Changes.
An image won't showThe path doesn't match the file. Check spelling, capitalization (Hero.jpg ≠ hero.jpg), the folder name, and that the file was actually copied into images/. The admin's live preview thumbnails are a quick truth-test: if the thumbnail is blank, the path is wrong.
The admin looks out of dateYou're on a different browser or computer than last time — autosave doesn't travel. Drag your live config.js into the import zone at the top of the admin.
A popup won't appear when testingPopups (and some other config-driven features) don't run when opening index.html directly from a folder. Test them via Save & Preview or on the live site.
Contact form does nothingThe Formspree Endpoint is blank or mistyped. It should look like https://formspree.io/f/xxxxx. Create a free form at formspree.io and paste the exact URL.
A custom Google Font isn't applyingThe name in the custom font field must match Google Fonts exactly — spaces and capital letters included (e.g., Cormorant Garamond).
“I deleted something by accident”Drag your most recent backup config.js into the import zone. This is why the dated backups exist. If you only exported the section as JSON, re-import it via the section builder's ⊕ Import button.
A page vanished from the menuCheck the page's “Show in navigation” switch, its hide-from-main-nav / hide-from-footer settings, and Members & Access (a locked page shows only a lock icon for guests).
Old page link broken after renamingThe URL slug changed. Restore the old slug in Page Settings, or accept the new address and update anywhere you shared the old one.
Maps aren't interactiveInteractive embedded maps need a Google Maps API key in Business Details (with the Maps Embed API enabled). Without one, use the “Link-out button” map mode instead.
The universal reset

When in doubt: import the live config.js into the admin, make your change, preview, export, replace. That loop recovers from almost any confusion — because the file in your website folder is always the single source of truth.
